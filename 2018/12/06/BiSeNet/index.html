论文阅读:BiSeNet

发表于 2018-12-06 | 更新于 2018-12-10 | 分类于 计算机视觉
本文字数： 2.7k | 阅读时长 ≈ 2 分钟

BiSeNet提出双向分割网络，包含两个部分：Spatial Path(SP)和 Context Path(CP).这两个部分分别用来解决空间信息缺失和感受野缩小的问题。

语义分割要求有丰富的空间信息和感受野。然而目前大多数的实时语义分割方法大多通过妥协空间分辨率以达到实时的要求，导致性能较差。旷世科技提出一种双向分割网络Bilateral Segmentation Network (BiSeNet)，不仅实现了实时语义分割， 还在特征融合模块和注意力优化模块的帮助下，把语义分割的性能推进到一个新高度。

背景

语义分割是计算机视觉领域的一项基本任务，旨在为一张图片上的每一个像素打上语义标签，可应用语增强现实、自动驾驶、安防监控等领域，这些应用要求能够快速响应。最近，实时语义分割的算法主要有以下三种方法：

1、通过裁剪或resize限定输入的大小，以降低计算复杂度。尽管这种方法简单有效，但是空间细节信息的损失尤其是边界信息的损失，导致度量和可视化精度下降。

2、通过减少网络通道数加快处理速度，尤其是在骨干网络的早期阶段，但是这会弱化空间信息。

3、为追求及其紧凑的框架而丢弃模型的最后阶段(比如ENet)，这种方法的缺点也很明显：ENet丢弃了模型最后阶段的下采样，模型的感受野不足以涵盖大物体，导致判别能力较差。

总之，这三种方法都是折中精度以求速度，难以付诸实践。如图一(a)所示。

为了补救上文提到的空间细节损失，研究者广泛利用U型结构，U型结构逐渐增加空间分辨率并恢复一些丢失的细节。然而，这种结构有两个缺点：1.完整的U型结构由于高分辨率的特征图而引入额外计算导致速度降低2.更重要的是，大多数通过裁剪或减少通道而损失的信息无法通过引入浅层而轻易恢复。换言之，U型结构最多是一个备选方法，而不是最终解决方案。如图一(b)所示

设计思想

基于上述观察，提出的双向分割网络Bilateral Segmentation Network (BiSeNet)包含两部分：Spatial Path (SP) and Context Path (CP)。这两个部分分别用来解决空间信息缺失和感受野缩小的问题。对于Spatial Path,本文只叠加三个卷积层以获得1/8特征图，保留了丰富的空间细节。对于Context Path，在Xeption尾部附加一个全局平均池化，其中感受野是骨干网络的最大值。如图一(c)所示。

为了在不损失速度的情况下追求更高的准确率，我们也研究了两个模块的融合和最后预测的优化。并分别提出了特征融合模块Feature Fusion Module (FFM)和注意力优化模块Attention Refinement Module (ARM)。

BiSeNet

本节中，首先详细介绍了BiSeNet网络及其两路网络 Spatial Path 和 Context Path；接着， 分别证明其有效性；最后，展示如何利用特征融合模块整合这两路网络的特征以及BiSeNet的整体架构。

Spatial Path

在语义分割任务中， 空间分辨率和感受野很难两全，尤其是在实时语义分割的情况下，现有的方法通常是利用小的输入图像或者轻量主干模型实现加速。但是小图像相较于原图像缺失了很多空间信息， 而轻量级模型则由于裁剪通道而损失了空间信息。

基于这一观察，本文提出的Spatial Path以保留原始输入图像的空间尺度，并编码丰富空间信息。Spatial Path包含三层，每层包含一个步长为2的卷积，随后是批归一化层和RelU层。因此，该路网络可以输出相当于原始图像1/8大小的特征图。由于大尺寸的特征图，编码丰富的空间特征。图二显示了结构细节

图二：BiSeNet概览

Context Path

在语义分割任务中，感受野对于性能表现至关重要。为增大感受野，一些方法利用金字塔池化模块，金字塔型空洞池化(ASPP)或者“large kernel”，但是这些操作比较耗费计算和内存， 导致速度较慢。出于较大感受野和较高计算效率兼得的考量， 本文提出Context Path， 它充分利用轻量级模型与全局平均池化以提供大感受野。轻量级模型中，本文借助 U 形结构融合最后两个阶段的特征，但这不是一个完整的 U 形结构。图 2(c) 全面展示了 Context Path。

网络架构

在 Spatial Path 和 Context Path 的基础上，本文提出 BiSeNet，以实现实时语义分割，如图 2(a) 所示。本文把预训练的 Xception 作为 Context Path 的 backbone，把带有步幅的三个卷积层作为 Spatial Path；接着本文融合这两个组件的输出特征以做出最后预测，它可以同时实现实时性能与高精度。

首先，本文聚焦于实际的计算方面。尽管 Spatial Path 有大感受野，但只有三个卷积层。因此，它并非计算密集的。对于 Context Path，本文借助一个轻量级模型快速下采样。进而，这两个组件并行计算，极大地提升了效率。第二，本文还讨论了网络的精度方面。Spatial Path 编码丰富的空间信息，Context Path 提供大感受野，两者相辅相成，从而实现更高性能。

特征融合模块：在特征表示的层面上，两路网络的特征并不相同。因此不能简单地加权这些特征。由 Spatial Path 捕获的空间信息编码了绝大多数的丰富细节信息。而 Context Path 的输出特征主要编码语境信息。换言之，Spatial Path 的输出特征是低层级的，Context Path 的输出特征是高层级的。因此，本文提出一个独特的特征融合模块以融合这些特征。

在特征的不同层级给定的情况下，本文首先连接 Spatial Path 和 Context Path 的输出特征；接着，通过批归一化平衡特征的尺度。下一步，像 SENet 一样，把相连接的特征池化为一个特征向量，并计算一个权重向量。这一权重向量可以重新加权特征，起到特征选择和结合的作用。图 2(c) 展示了这一设计的细节。

损失函数：本文通过辅助损失函数监督模型的训练，通过主损失函数监督整个 BiSeNet 的输出。另外，还通过添加两个特殊的辅助损失函数监督 Context Path 的输出，就像多层监督一样。上述所有损失函数都是 Softmax。最后借助参数 α 以平衡主损失函数与辅助损失函数的权重。
